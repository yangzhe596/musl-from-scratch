# Day 11 - 十进制与十六进制整数转字符串

## 今天的目标

- 实现十进制和十六进制整数转字符串
- 处理 `%d` 的负数
- 解决输出顺序问题

## 预计投入

- 15 分钟：思考“从低位到高位”怎么输出
- 30 分钟：实现转换函数
- 15 分钟：写最小测试

## 今日任务

1. 实现两个方向中的一个：
   - 先写入临时缓冲，再反转
   - 从缓冲末尾往前填
2. 至少实现：

```c
static char *utoa10(unsigned int x, char *end);
static char *utoa16(unsigned int x, char *end);
static char *itoa10(int x, char *end);
```

3. 测试：
   - `0`
   - 正数
   - 负数
   - 一个较大的十六进制值

## 重点理解

- `%d` 和 `%x` 的本质都是“基数转换”
- `INT_MIN` 是一个值得特别留意的边界

## 验证标准

- 你能解释自己为什么选择“反转”或“尾部回填”
- `0` 不会输出成空串
