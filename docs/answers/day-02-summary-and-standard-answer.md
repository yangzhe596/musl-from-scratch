# Day 02 - 学习总结与标准答案

## 今日是否完成

Day 2 可以视为完成。

今天的目标不是写代码，而是通过观察一个普通 ELF 程序，确认程序真实入口、CRT 启动代码和 `main` 之间的关系，并为后续手写 `_start` 做认知准备。

## 今日总结

今天最重要的结论是：程序不是从 `main` 开始执行，而是从 ELF 头中记录的入口地址开始执行。这个入口地址通常对应 `_start`。

在普通 glibc 程序中，`_start` 通常不是你自己写的，而是由 CRT 启动代码提供。它接住内核交来的控制流，做最小启动准备，然后把控制权交给 libc 的启动逻辑。你在反汇编中已经看到，`_start` 并不是直接调用 `main`，而是把 `main` 的地址传给 `__libc_start_main`，再由后者去完成后续工作。

这说明平时写 C 程序时之所以感觉不到 `_start` 的存在，不是因为它不存在，而是因为工具链已经自动替你接入了 CRT 和 libc。你只需要写 `main`，但真正的程序入口仍然是 `_start`。

这也解释了为什么只有一个 `int main(void) { return 0; }` 通常不能在没有 CRT 的情况下独立运行：因为 `main` 不是程序真实入口，没有 CRT，就没有 `_start` 来接住内核控制权，也没有人负责调用 `main` 和处理退出路径。

## 标准答案

### 1. 如果 ELF 入口点是 `_start`，为什么平时写 C 程序时几乎感觉不到它的存在？

标准答案：

因为在普通编译和链接流程中，工具链会自动接入 CRT 启动代码和 libc。CRT 中通常已经提供了 `_start`，并负责在启动阶段完成最小准备后调用 `main`。因此程序实际仍然从 `_start` 开始，只是这层过程平时被工具链隐藏了。

### 2. 什么是 CRT？

标准答案：

CRT 可以先理解为 C Runtime Startup Code，也就是 C 程序启动时最前面的那层运行时代码。它通常负责：

- 提供程序真实入口 `_start`
- 接住内核交来的执行流
- 做最小启动准备
- 调用 `main`
- 在 `main` 返回后走正确的退出路径

在这个项目里，我们会自己手写一个极简版启动路径，用来替代宿主环境自动提供的 CRT。

### 3. 为什么只有一个 `int main(void) { return 0; }` 通常不能独立运行？

标准答案：

因为 `main` 不是程序真实入口。没有 CRT 时，程序缺少真实入口 `_start`，也缺少调用 `main` 和处理退出流程的运行时代码，因此不能像普通程序那样独立运行。

### 4. 你今天观察到的 ELF 程序说明了什么？

标准答案：

通过 `readelf` 和 `objdump` 可以确认：

- ELF 文件中存在明确的入口地址
- 符号表中同时存在 `_start` 和 `main`
- `_start` 才是程序真实入口
- 在普通 glibc 程序里，`_start` 不一定直接调用 `main`，而是常常先调用 `__libc_start_main`

### 5. 反汇编中的 `_start` 最重要的观察结论是什么？

标准答案：

今天最重要的不是逐条看懂每条汇编，而是看懂控制流关系。你观察到：

```asm
lea ... <main>
call __libc_start_main
```

这说明：

- `_start` 会准备 `main` 的地址
- `_start` 把后续启动工作交给 `__libc_start_main`
- 普通 glibc 程序的路径更准确地说是：
  - 内核 -> `_start`
  - `_start` -> `__libc_start_main`
  - `__libc_start_main` -> `main`

### 6. 为什么 `_start` 之后还需要一条明确的退出路径？

标准答案：

因为 `main` 返回的整数不能凭空消失，必须由运行时接住并交还给内核，作为进程退出码。在普通环境中，这件事由 CRT/libc 帮你完成；在本项目中，后续将由我们自己写的 `_start` 通过 `exit` syscall 完成。

### 7. 我们这个项目会如何简化普通程序的启动路径？

标准答案：

我们不会照搬 glibc 的完整启动链，而是实现一个教学版极简模型：

- 自己写 `_start`
- 不引入 `__libc_start_main`
- 直接调用自己的 C 入口函数
- 在 C 入口返回后，通过 `exit` syscall 显式结束进程

## 今日最小知识闭环

如果今天只记住 6 句话，应该记住这些：

- ELF 程序从入口地址开始执行，不是从 `main` 开始
- 入口地址通常对应 `_start`
- `_start` 通常由 CRT 启动代码提供
- 普通 glibc 程序里，`_start` 经常会调用 `__libc_start_main`
- `main` 是运行时调用的程序主体逻辑入口，不是真实入口
- 我们后续要自己手写一个极简 `_start`
