## 今日总结

今天的重点是实现整数转字符串，这是 `printf` 中 `%d` 和 `%x` 的基础。

在实现方面，我们选择了"尾部回填"策略：
- `utoa10(unsigned int x, char *end)`：从缓冲区末尾往前填十进制数字
- `utoa16(unsigned int x, char *end)`：从缓冲区末尾往前填十六进制数字
- `itoa10(int x, char *end)`：处理负号后委托给 `utoa10`

关键设计：
1. 为什么从后往前填：数字是从低位到高位生成的，从后往前填正好是正确顺序，不需要反转
2. `INT_MIN` 的处理：用 `~(unsigned int)x + 1u` 做二进制取绝对值，避免 `-INT_MIN` 溢出
3. `itoa10` 和 `utoa10` 分层：`utoa10` 是核心（纯数学转换），`itoa10` 是薄包装（处理符号）

踩的坑：
- 最初 `itoa10` 对 `INT_MIN` 返回字符串常量，导致 `put_int` 的打印逻辑越界
- 修复：统一走 `utoa10` 的缓冲区回填逻辑

## 标准答案

### 1. 为什么从后往前填比反转好？

标准答案：

从后往前填的优势：
- 数字是从低位到高位生成的（`x % 10`），从后往前填正好是正确顺序
- 不需要额外的反转步骤，减少一次遍历
- 只需要一个缓冲区，不需要临时空间

反转方案需要：先从前往后填，再反转。多了一次遍历和额外逻辑。

### 2. INT_MIN 为什么特殊？

标准答案：

`INT_MIN`（-2147483648）的绝对值（2147483648）超过了 `int` 的最大值（2147483647），所以 `-INT_MIN` 会导致有符号整数溢出，这是未定义行为。

解决方法：用位运算 `~(unsigned int)x + 1u` 做二进制取绝对值，绕过有符号溢出问题。这利用了二进制补码的特性。

### 3. 为什么 itoa10 和 utoa10 要分层？

标准答案：

分层的原因：
- `utoa10` 是核心：纯粹的"无符号数字转字符串"，不关心正负
- `itoa10` 是薄包装：处理负号后委托给 `utoa10`

这样的好处：
- `%d`（有符号）用 `itoa10`
- `%u`（无符号）直接用 `utoa10`
- `%x` 用 `utoa16`

每个调用点选最合适的底层函数，不需要在一个函数里用标志位区分有符号/无符号。
