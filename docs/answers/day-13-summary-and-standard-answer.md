## 标准答案

### 1. 为什么 musl 不会把所有逻辑都塞进 printf.c？

标准答案：

musl 把 `printf` 做成 `vfprintf` 的薄包装，原因是：
- `fprintf`、`sprintf`、`snprintf` 都需要同样的格式解析逻辑
- 把核心逻辑放在 `vfprintf` 里，所有 printf 变体都能复用
- `printf.c` 只负责把 `va_list` 转换后委托给 `vfprintf`

这是经典的"委托模式"：薄包装 + 核心实现分离。

### 2. 当前实现距离工业级 printf 还差什么？

标准答案：

| 方面 | 当前实现 | 工业级 |
|------|----------|--------|
| 格式符 | `%s` `%d` `%x` `%%` | 完整 C99 格式符集 |
| 长度修饰符 | 无 | `%ld` `%lld` `%hd` 等 |
| 宽度/精度 | 无 | `%8d` `%.3f` 等 |
| 标志 | 无 | `%-` `%+` `%0` `%#` 等 |
| 浮点 | 无 | `%f` `%e` `%g` |
| 输出后端 | 直接 sys_write | FILE 抽象，支持缓冲 |
| 返回值 | 不完整 | 精确的字符计数 |

### 3. 如果要支持 %p，会复用 %x 的哪部分逻辑？

标准答案：

`%p` 输出指针地址，本质上就是输出一个 `unsigned long` 的十六进制表示。可以复用：
- `utoa16` 的转换逻辑
- `put_hex` 的输出逻辑

区别在于：
- `%p` 通常带 `0x` 前缀
- `%p` 用 `va_arg(ap, void *)`，然后转成 `unsigned long`
- `%p` 通常输出固定宽度（16 位，补前导零）
