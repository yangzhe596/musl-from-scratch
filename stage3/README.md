# Stage 3 - 格式化输出起步

## 当前目标

- 从 `printf` 的复杂度里先把“输出后端”拆出来
- 用最小 `put_char` / `put_str` 完成直接输出
- 用 `va_list` 实现一个最小可变参数实验函数 `demo`
- 今天不实现格式解析，先把框架跑通

## 文件说明

| 文件 | 职责 |
|------|------|
| `printf.h` | 最小输出接口与可变参数实验函数声明 |
| `printf.c` | `put_char`、`put_str`、`demo` 实现 |
| `main.c` | 当天测试入口 |

依赖 `stage1/` 的 `start.S` 和 `syscall.c`。

## 今日设计

### 输出策略

采用“**边解析边输出**”：
- 每输出一个字符就直接调用 `sys_write`
- 优点是实现简单、调试直观
- 后续 `printf` 实现时再考虑缓冲优化

### va_list 实验

`demo` 的设计目的是：
- 确认 `va_start` / `va_arg` / `va_end` 能正常工作
- 练习可变参数遍历终止方式
- 把“参数解析”与“输出打印”先解耦

## 编译与运行

```bash
make
make run
```

## 验证点

- `put_char` 能逐字符输出
- `put_str` 能输出普通字符串，空指针不崩
- `demo` 能正确打印 tag 和可变字符串列表
