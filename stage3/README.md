# Stage 3 - 格式化输出

## 实现功能

- `put_char` — 逐字符输出（每次调用 `sys_write`）
- `put_str` — 字符串输出，空指针安全
- `put_int` — 有符号十进制整数输出
- `put_hex` — 无符号十六进制整数输出（小写）
- `demo` — `va_list` 实验函数（Day 10 遗留）
- `mini_printf` — 最小格式化输出

## mini_printf 格式支持

### 已支持

| 格式 | 说明 | va_arg 类型 |
|------|------|-------------|
| `%s` | 字符串 | `const char *` |
| `%d` | 有符号十进制 | `int` |
| `%x` | 无符号十六进制（小写） | `unsigned int` |
| `%%` | 输出字面 `%` | 无 |

### 未支持

| 格式 | 说明 |
|------|------|
| `%u` | 无符号十进制 |
| `%ld` / `%lx` / `%lu` | long 修饰符 |
| `%c` | 单字符 |
| `%p` | 指针地址 |
| `%o` | 八进制 |
| `%X` | 大写十六进制 |
| `%8d` / `%-8d` / `%08x` | 宽度、对齐、填充 |

### 未知格式处理

遇到不认识的格式符（如 `%q`），原样输出 `%q`，不崩溃。

## 与 musl 的关键差异

| 方面 | 本实现 | musl |
|------|--------|------|
| 输出后端 | 直接 `sys_write` 每个字符 | 抽象 `FILE *`，支持缓冲/无缓冲/行缓冲 |
| 格式解析 | 简单 switch-case | 多层状态机，支持标志/宽度/精度/长度修饰符 |
| 类型处理 | `int` / `unsigned int` | `intmax_t` / `size_t` 等统一类型 |
| 分层 | `mini_printf` 做所有事 | `printf` → `vfprintf`，可复用核心逻辑 |

**真正复杂的是格式解析的状态机，不是输出本身。**
